This section demos .container class, this is equivalent to .container-sm. This means that at the smallest screen size the container is 100% wide. As the screen size expands and hits the small breakpoint, i.e., it is 576px then the container is no longer 100%, it is centered on the screen with some set width. At -sm it is 540px, at -md it becomes 720px and so on.

This section demos .container-lg class. This means that as long as the screen width is less than the -lg size, i.e., 992px, the container will be 100% wide. As soon as the screen expands to -lg size, the container is no longer 100%, it is cenetered with some set width. At -lg it is set to 960px, when the screen expands to -xl width, i.e., 1200px, the container width will be 1140px, and so on.

This section demonstrates the responsive column. The concept is similar to .containers, where there is a some default behavior for screens that narrower than the specified width, and then as the screen width expands to pre-defined breakpoints, -sm, -lg, -xl, etc. the behavior changes.

In case of columns, in narrower screens the columns are horizontally stacked on top of each other. When the screen hits the specified breakpoint, they start behaviing like real columns. This is how the first row behaves. For screen narrower than the -lg size the columns are stacked on top of each other, for screen sizes -lg and wider, the first column is 3 units, the second column is 4 units wide, and the third column is 5 units.

The second row is a bit more sophisticated. The first column is 3 units for screens -lg or wider, but 4 units for -md screens, and beyond that will be horizontally stacked.

The third row demonstrates variable width column where the column width adjusts to the size of the content. There is more going on here than just this. The first column is auto-width for smaller screens, for -lg and wider screens, the width is 2 units. The second column is variable width for -md screens. The third column is always auto-width. The fourth row has the same configuration but with smaller content.
